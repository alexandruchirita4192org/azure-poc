# Roadmap

## From Foundations to Certification

```text
AI Foundations
    ↓
.NET 10 Modernization
    ↓
Azure / AWS Deployment PoCs
    ↓
AI-200 Direction
```

### What each phase means

- **AI Foundations**: build shared vocabulary around prompts, retrieval, evaluation, safety, and practical AI system design.
- **.NET 10 Modernization**: move the codebase and supporting patterns to the current .NET platform baseline.
- **Azure / AWS Deployment PoCs**: prove the same workload on both clouds to clarify portability, tradeoffs, and service mapping.
- **AI-200 Direction**: decide whether the next learning path should target applied Microsoft AI delivery and exam readiness.

### Active Recall: AZ-204 Weak Areas

Use these as weekly self-check questions:

- When do I choose **Managed Identity** over connection strings, and how do I wire it in App Service, Functions, and Container Apps?
- How do I secure **API Management** with subscriptions, policies, and backend key injection?
- What is the difference between **Service Bus**, **Event Grid**, and **Storage Queues**, and when would I use each?
- How do I configure **Azure Functions** triggers and bindings for Event Grid and queue-driven workflows?
- How do I choose between **App Service**, **Container Apps**, and **Functions** for a .NET workload?
- How do I store secrets in **Key Vault** and reference them safely from Azure services?
- How do I observe a distributed app with **Application Insights** and dependency telemetry?
- How do I deploy repeatably with **Bicep** and GitHub Actions?
- What are the common failure modes when integrating **Azure SQL**, **Cosmos DB**, and **Blob Storage**?
- What would I need to explain if asked to migrate this workload to AWS services?

### Slide Footer

- Goal: turn platform knowledge into repeatable delivery skill.
- Output: one modernized .NET path, two deployment PoCs, one exam direction.
